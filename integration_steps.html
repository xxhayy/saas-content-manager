Assets Page — Integration Steps

A beginner-friendly, step-by-step guide to building the Assets feature from scratch.

20 Steps 6 Phases 14 Files AirOne Studio
1

External Setup (You Do This Manually)

One-time tasks done outside the code editor
01

Create an ImageKit Account

ImageKit is where we'll store all uploaded images (originals + AI-processed versions).

  1. Go to imagekit.io and sign up (free tier is fine)
  2. Once logged in, go to Settings → Developer Options → API Keys
  3. You'll see 3 values: URL Endpoint, Public Key, Private Key
  4. Copy all 3 — you'll need them in Step 3
02

Get Your kie.ai API Key

You already have this! It's the KIE_API_KEY from your earlier project. Make sure you know where it is.

03

Add Keys to Your .env File

Open your .env file in the project root and add these lines at the bottom:

# ImageKit
NEXT_PUBLIC_IMAGEKIT_PUBLIC_KEY=paste_your_public_key_here
IMAGEKIT_PRIVATE_KEY=paste_your_private_key_here
NEXT_PUBLIC_IMAGEKIT_URL_ENDPOINT=https://ik.imagekit.io/your_id

# kie.ai
KIE_AI_API_KEY=paste_your_kie_api_key_here

# Internal Secret (type any random long string, like a password)
INTERNAL_API_SECRET=make_up_a_random_string_here_abc123xyz
💡 Tip

The INTERNAL_API_SECRET is a password you make up yourself to protect your internal API route. Just type any random string — it doesn't come from any service.

2

Database (Adding the Asset Table)

Tell the database about the new Asset structure
04

Update the Prisma Schema

Edit prisma/schema.prisma to add the following:

  • An AssetStatus enum: PROCESSING, COMPLETED, FAILED
  • An AssetCategory enum: FURNITURE, COMMERCE_PRODUCT, AVATAR
  • The full Asset model with all fields (name, blueprint, urls, status, retryCount…)
  • A new assets Asset[] relation on the existing User model
05

Push Schema to Database

Run these two commands to sync the new schema to your Supabase database:

npx prisma db push
npx prisma generate
ℹ️ Note

db push creates the new asset table in Supabase PostgreSQL. generate updates the TypeScript types so your code knows about the new model.

3

Server-Side Services (The Backend Brain)

Create helper modules for ImageKit and kie.ai
06

Create the ImageKit Service

Create src/server/imagekit.ts that:

  • Configures the ImageKit SDK with your keys from .env
  • Exports a helper function to upload images to ImageKit folders
07

Create the kie.ai Service

Create src/server/kie-ai.ts that contains:

  • submitTask() — sends an image + prompt to https://api.kie.ai/api/v1/jobs/createTask
  • pollTask() — checks https://api.kie.ai/api/v1/jobs/recordInfo every 4 seconds until done
  • Category prompt templates — different prompts for Furniture, Commerce Product, and Avatar
08

Update Environment Validation

Edit src/env.js to add validation rules for the new environment variables. This makes the app warn you at startup if any API keys are missing.

4

API Routes (The Endpoints)

5 API routes that power the entire backend pipeline
09

Create the ImageKit Auth Route

Create src/app/api/imagekit-auth/route.ts:

  • Returns auth tokens so the browser can upload images directly to ImageKit
  • Required because ImageKit needs a server-signed auth for secure uploads
10

Create the Upload Route

Create src/app/api/assets/upload/route.ts:

  • Receives a list of ImageKit URLs + the selected category
  • Creates one Asset database record per image (status: PROCESSING)
  • Fires off the processing chain in the background
  • Returns asset IDs immediately — the user doesn't wait
11

Create the Process-Next Route

Create src/app/api/assets/process-next/route.ts — the heart of the system:

  • Finds the oldest PROCESSING asset
  • Sends it to kie.ai → waits for result → uploads clean image to ImageKit → updates DB
  • If it fails, retries up to 3 times before marking as FAILED
  • If there are more assets to process, calls itself again automatically
  • Secured with INTERNAL_API_SECRET so only our server can trigger it
12

Create the Assets List Route

Create src/app/api/assets/route.ts:

  • Returns all assets for the logged-in user
  • Supports filtering by category
  • Used by the frontend gallery to display and poll for updates
13

Create the Recent Assets Route

Create src/app/api/assets/recent/route.ts:

  • Returns the 5 most recent assets
  • Used by the dashboard home page for notifications
5

Frontend (What the User Sees)

5 components and pages that make up the Assets UI
14

Build the Asset Card Component

Create src/components/assets/asset-card.tsx — a card with 3 visual states:

  • ⏳ Processing — spinner + "Processing..." text overlay
  • ✅ Completed — clean white-background image + auto-generated name + category badge
  • ❌ Failed — error message + "Retry" button
15

Build the Upload Zone Component

Create src/components/assets/asset-upload-zone.tsx — a popup/dialog with:

  • Category selector (Furniture / Commerce Product / Avatar) — required before upload
  • Drag-and-drop area for images (max 10 images)
  • Preview thumbnails of selected files
  • "Upload" button that triggers the entire pipeline
16

Build the Assets Page

Create src/app/(dashboard)/assets/page.tsx — the main page at /dashboard/assets:

  • 3 category tabs: Furniture | Commerce Product | Avatar
  • Grid of asset cards filtered by the selected tab
  • "Upload Assets" button in the header
  • Auto-polls the server every 5 seconds while assets are processing
17

Update the Dashboard Page

Edit src/app/(dashboard)/dashboard/page.tsx:

  • Add a "Recent Assets" section showing the 5 most recent assets
  • Show each asset with its current status badge
  • Acts as a notification center for completed processing
18

Link Assets in the Sidebar

Confirm the "Assets" link in the sidebar navigation (app-sidebar.tsx) points to /dashboard/assets.

6

Test Everything

Verify the feature works end-to-end
19

Test the Happy Path

  1. Navigate to /dashboard/assets
  2. Click "Upload Assets" and select Furniture as the category
  3. Drop in 2–3 test images → click Upload
  4. See "Submitted" confirmation — cards show spinners
  5. Navigate away to /dashboard
  6. Wait ~60 seconds, return to /dashboard/assets
  7. ✅ All images show clean white backgrounds + auto-generated names
  8. ✅ Dashboard shows recent asset notifications
20

Test Edge Cases

  • Upload more than 10 images → should be blocked with an error
  • Upload a non-image file (e.g. a PDF) → should be rejected
  • Check "Commerce Product" tab is empty when you only uploaded to "Furniture"
  • Verify that an asset stuck as FAILED shows a Retry button
