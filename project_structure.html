📁 Root Directory

  • `.env` / `.env.example`: Environment variable configurations (API keys, database URLs).
  • `next.config.js`: Next.js configuration (images, experimental features).
  • `package.json`: Project dependencies, scripts, and metadata.
  • `prisma/`: Database management folder.
  • `public/`: Static assets like favicons and public images.
  • `src/`: The main source code directory.
  • `tailwind.config.ts`: Tailwind CSS theme and plugin configuration.
  • `tsconfig.json`: TypeScript compiler options and path mappings.
  • `eslint.config.js`: Linting rules for code quality.
  • `postcss.config.js`: CSS processing configuration.

📁 prisma/

The data layer of the application.

  • `schema.prisma`: The single source of truth for the database schema (Models: User, Session, Account, Asset).
  • `migrations/`: Automatically generated SQL files that track database schema changes.

📁 src/app/

The core of the Next.js App Router architecture.

Routing Groups

  • `(auth)/`: Layout and dynamic routing for Better Auth authentication.
  • `(dashboard)/`: Protected routes including main overview and asset gallery.
  • `layout.tsx`: Root HTML structure and global providers.
  • `page.tsx`: Application landing page.

API Layer

  • `api/auth/`: Server-side handler for Better Auth.
  • `api/upload-auth/`: Secure ImageKit authentication endpoint.
  • `api/assets/`: Sequential background workers and upload handlers.

📁 src/components/

  • `assets/`: UI for Asset cards and the specialized upload zone.
  • `ui/`: Core design system built with shadcn/ui.
  • `app-sidebar.tsx`: Dynamic navigation sidebar.
  • `providers.tsx`: Client-side context container.

📁 src/server/

  • `db.ts`: Singleton Prisma initialization.
  • `imagekit.ts`: ImageKit storage service integration.
  • `kie-ai.ts`: AI processing service (Nano-Banana-2) with polling logic.
  • `better-auth/`: Auth plugin and config directory.

📁 Utility Folders

src/lib/

  • `auth.ts`: Better Auth server instance.
  • `auth-client.ts`: Better Auth client utilities.
  • `utils.ts`: Tailwind class merging helper.

src/hooks/ & styles/

  • `use-mobile.ts`: Responsive viewport detection.
  • `globals.css`: Global CSS variables and Tailwind layers.
